## AWS 루트 사용자
AWS 계정을 처음 생성할 때는 계정의 모든 AWS 서비스 및 리소스에 대해 완전한 액세스 권한을 갖게 된다. 이를 AWS 루트 사용자라고 하며, 게정을 생성할 때 사용한 이메일 주소 및 암호로 로그인하여 액세스한다.

## AWS 루트 사용자 자격 증명
AWS 루트 사용자에게는 2가지 자격 증명 집합이 연결되어 있다. 자격 증명 집합 중 하나는 계정을 만드는데 사용되는 이메일 주소와 암호이다. 이를 통해 AWS Management Console에 액세스할 수 있다. 두 번째 자격 증명 집합은 액세스 키라고 한다. 이 키를 사용해서 AWS CLI 또는 AWS API에서 프로그래밍 방식으로 요청을 수행할 수 있다.

액세스키는 액세스 키 ID와 비밀 액세스 키로 구성된다.

사용자 이름 및 암호 조합과 마찬가지로, AWS CLI 또는 AWS API를 통해 요청을 인증하려면 액세스 키 ID와 비밀 액세스 키를 함께 사용해야 한다. 액세스 키는 이메일 주소 및 암호와 동일한 보안 기능을 사용하여 관리해야 한다.

AWS 계정 루트 사용자에 대한 액세스 키가 없다면 필요할 때까지 만들지 마라.

## AWS 루트 사용자 모범 사례
루트 사용자는 결제 및 개인 정보를 비롯하여 계정의 모든 AWS 서비스 및 리소스에 대한 전체 액세스 권한을 갖는다. 따라서 루트 사용자와 관련된  자격 증명을 안전하게 보관하고 일상 작업에는 루트 사용자를 사용하지 않아야 한다.

모범 사례
- 루트 사용자에 강력한 암호를 설정한다.
- 루트 사용자에 다중 인증(MFA)를 사용한다.
- 루트 사용자 암호나 액세스 키를 누구와도 공유하지 않는다.
- 루트 사용자와 연결된 액세스 키는 사용 중지하거나 삭제한다.
- 관리 작업이나 일상 작업용으로 IAM 사용자를 생성한다.

## 다중 인증
AWS 계정을 생성하고 처음 계정에 로그인할 때 1단계 인증을 사용한다. 1단계 인증은 가장 단순하고 가장 일반적인 인증 형식이다. 인증 방법이 하나만 있으면 된다. 이 경우 사용자 이름과 암호를 사용하여 AWS 루트 사용자로 인증한다. 1단계 인증의 다른 형태로는 보안 핀 또는 보안 토큰이 있다.

그러나 사용자의 암호는 추측하기 쉬울 때가 많다. 따라서 다중 인증(MFA)을 사용하는 것이 원치 않은 계정 액세스를 방지하는 데 중요하다.

MFA에서는 2개 이상의 인증 방법을 사용하여 자격 증명을 확인해야 한다. MFA에서는 사용하는 3가지 정보는 알고있는 정보(사용자 이름과 암호, 핀 번호), 가지고 있는 정보(하드웨어 디바이스 또는 모바일 앱에서 생성된 일회용 패스 코드), 신원 정보(지문, 얼굴 인식 기술 등)이 있다.

이 정보의 조합으로 시스템이 계정 액에스에 대해 계층화된 접근법을 제공할 수 있다. 따라서 악의적인 행위자가 첫 번째 인증 방법을 해킹하더라도 두 번째 인증 방법이 또 다른 수준의 보안을 제공한다. 이러한 추가 보안 계층은 가장 중요한 계정을 보호하는 데 도움이 될 수 있으므로 AWS 루트 사용자에 MFA를 활성화해야 한다.

## AWS 기반 MFA
루트 사용자에게 MFA를 활성화하는 경우, 알고 있는 정보 범주와 가지고 있는 정보 범주 모드에서 식별 정보를 제시해야 한다. 사용자가 입력하는 식별 정보의 첫 번째는 이메일 및 암호 조합이다. 두번째 정보는 MFA 디바이스에서 제공하는 임시 숫자 코드이다.

MFA를 사용하면 사용자가 일반 로그인 자격 증명 외에 지원되는 MFA 메커니즘을 사용해야 하므로 보안 계층이 강화된다. AWS 루트 사용가 계정에서 MFA를 활성화 하는 것이 AWS 모범 사례이다.

## 지원되는 MFA 디바이스
AWS는 가상 MFA 디바이스, 시간 제한이 있는 하드웨어 일회용 암호(TOTP) 및 FIDO 보안 키와 같은 다양한 MFA 메커니즘을 지원한다.

| ------------ | ------------------------------------------------------------------------------------------------------------------------- | ---------------------------------------------------------------------------------------------------------------------------- |
| 가상 MFA       | 휴대폰이나 기타 디바이스에서 실행되는 소프트웨어 앱, 일회용 암호를 제공한다.<br>보안되지 않은 모바일 디바이스에서 실행될 수 있으므로 하드웨어 또는 FIDO 보안 키와 동일한 수준의 보안을 제공하지 않을 수 있다. | Twilio Authy Authenticator, Duo Mobile, LastPass, Authenticator, Microsoft Authenticator, Google Authenticator, Symantec VIP |
