![[image 3.png]]AWS 클라우드로 작업할 때 보안 및 규정 준수를 관리하는 것은 AWS와 고객의 공동 책임이다. 이러한 공동 책임을 나타내기 위해 AWS는 공동 책임 모델을 만들었다. 책임의 차이를 일반적으로 클라우드 자체의 보안, 클라우드 내부의 보안이라고 한다.

## AWS의 책임
![[image 4.png]]
AWS는 클라우드 자체의 보안을 담당한다. 즉, AWS는 AWS 클라우드에서 제공되는 서비스를 실행하는 인프라를 보호하고 보안을 유지한다.
AWS의 책임은 다음과 같다.
- AWS 리전, 가용 영역, 데이터 센터 보호, 건물의 물리적보안
- AWS 서비스를 실행하는 하드웨어, 소프트웨어, 네트워킹 구성 요소 관리

AWS의 책임 수준은 서비스에 따라 달라진다. AWS는 서비스를 2가지 범주로 분류한다.
- 인프라 서비스
	- 컴퓨팅 서비스 (예: Amazon EC2)
	- AWS는 기본 인프라 및 기초 서비스를 관리한다.
- 추상화 서비스
	- 고객의 관리가 거의 필요하지 않은 서비스 (예: Amazon S3)
	- AWS는 서버 측 암호화 및 데이터 보호 외에도 인프라 계층, 운영 체제 및 플랫폼을 운영한다.

## 고객의 책임
![[image 5.png]]
고객은 클라우드 내부의 보안을 책임진다. AWS 서비스를 이용할 때 고객은 데이터 보안을 보장할 뿐만 아니라 서비스 및 애플리케이션을 올바르게 구성할 책임이 있다.
고객의 책임 수준은 AWS 서비스에 따라 달라진다. 일부 서비스의 경우 필요한 모든 보안 구성 및 관리 작업을 고객이 수행해야 한다. 상대적으로 더 추상화된 서비스의 경우에는 고객이 데이터를 관리하고 고객의 리소스에 대한 엑세스를 제어하기만 하면 된다.

AWS 서비스의 2가지 범주에 따라 고객은 각 AWS 서비스에 대한 책임 수준을 판단할 수 있다.
- 인프라 서비스
	- 컴퓨팅 서비스 (예: Amazon EC2)
	- 고객은 고객 데이터를 암호화, 보호, 관리하는 것 외에도 운영 체제 및 애플리케이션 플랫폼을 제어한다.
- 추상화 서비스
	- 고객의 관리가 거의 필요하지 않은 서비스 (예: Amazon S3)
	- 고객은 고객 데이터를 관리하고, 데이터를 암호화하고, 네트워크 방화벽 및 백업을 통해 데이터를 보호할 책임이 있다.

서비스의 범주마다 책임 수준이 다르기에 고객은 어떤 AWS 서비스를 사용하는지 고려하고 각 서비스를 보호하는 데 필요한 책임 수준을 검토해야 한다. 또한 관련 법률 및 규정 외에도 AWS 공통 책임 모델이 IT 환경의 보안 표준과 어떻게 일치하는지 검토해야 한다.

핵심은 고객이 데이터를 완벽하게 제어하고 콘텐츠와 관련된 보안을 관리할 책임이 있다는 것이다.
- 데이터 주권 규정에 따라 AWS 리소스의 리전 선택
- 암호화 기법, 일정에 따른 백업 등 데이터 보호 메커니즘 구현
- 액세스 제어를 사용하여 데이터와 AWS 리소스에 액세스할 수 있는 사람을 제한
